wps_day31_memo

git 특강 4/4

git mv 명령

폴더 간 단순 이동이 아닌 git과 연동되었을 경우에 이동할 경우 앞에 git 붙여서 mv한다.

수업 코드 1

설명 - 혼자 작업할 때 일반적인 flow

git clone 통해서 만들고

git flow init으로 시작(만든 그 폴더에서)
git flow feature start

→만들거 만들고→
git add
git commit
git push로 origin의 feature 브랜치에 푸쉬

git flow feature finish

git flow release start
git flow release finish

git push -u origin develop
master로 checkout
git push -u origin master

수업 코드 2

설명 - 같이 작업할 때 일반적인 flow

PM의 repo를 내 계정으로 fork하고 clone해서 폴더를 만든다.

develop 브랜치를 만들어서 그곳에 checkout한 다음에 origin(PM의 것을 fork한 내 계정)의 develop 브랜치의 것을 pull한다.

main.py를 수정하고, 수정한 것을 add, commit하고, origin의 develop 브랜치에 push한다. 여기서 origin은 PM의 것을 fork한 내 계정의 해당 repo이다. 후에, 내 계정의 origin에서 pull request를 통해 PM한테 수정사항을 줄 수 있다.

PM의 remote 별칭을 또 하나 만들어본다. pmorigin으로 하였다.

나의 master 브랜치로 checkout 후에 PM의 master 브랜치에 있는 내용을 업데이트 하여 진행 상황을 최신 상태로 유지하자.
