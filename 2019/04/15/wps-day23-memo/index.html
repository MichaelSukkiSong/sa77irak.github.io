CS/network/network.pdf

tcp/ip 완벽 가이드(레퍼런스 책,추천책, Charles M. Kozierok)(노란 벌꿀 표지)

물리계층 : 눈에 보이는 것을 의미(LAN cable)
RJ45
Repeater

데이터 링크 계층
mac 주소

application: 전달받은 youtube 프로세스
transport: 소켓에 부여된 port번호를 관리. TCP와 UDP가 port를 관리.
Internet: IP. IP를 담당하는 얘가 internet protocol.
Network Interface: LAN에서의 mac주소를 찾는 것잉 이더넷 프로토콜

NIC의 mac주소: LAN안에서 컴퓨터를 찾을 때만 씀 끝.
IP주소: WAN상에서 특정 호스트(컴퓨터)의 주소를 의미.
port: 소켓에 부여된 일련 번호. 소켓은 process안에 있음.

이더넷 프로토콜(LAN영역 관련)
Destination MAC Address: 라우터 주소.LAN에서의 MAC 주소??
Soure MAC address: 내컴의 NIC
MTU: 한번에 보낼 수 있는 최대 데이터량이 1500바이트

traceroute google.com

ARP(Address Resolution Protocol): MAC 주소를 ->IP주소로 바꾸는 역할.
RARP(Reverse~): IP주소를 ->MAC주소로.

broadcast: LAN상에 모든 애들한테 패킷을 쏘는 것. MAC주소를 모를 때.

arp -a

IP

TTL(Time to Live): 패킷에 TTL넣어서 예)TTL5로 출발->라우터로 감->TTL4됨->->TTL0이 되면 최종 목적지 도착과 상관없이 패킷은 버려짐.
source address: 나의 주소 보내는 얘의 IP주소
destination address: 최종 목적지. 유투부 본사의 최종 ip주소

IP주소 구성
Network 주소: 라우터 주소.라우터를 찾는 주소. LAN을 찾을 떄 필요한 주소
Host 주소: LAN에 속해있는 특정 호스트(컴퓨터)를 찾는 주소.

subnet mask
어디까지가 서브넷을 포함한 주소고 어디까지가 호스트 주소인지 알 수 있음.

네트워크 주소: 호스트주소가 다 0이다.

IP

공인ip하나에 표 나와있는 것을 제외한 것중 하나를 배분 할 수 있는 것

공인 IP: 전세계 유일. ISP가 iptime하나에 대해 하나씩 줌. 존나 비쌈.
사설 IP: 공인 ip하나를 라우터에 걸어두고 사설망으로 써서 NAT를 통해 함. 바깥에서 나를 특정해서 보낼 수 없음.

그럼 IP주소는 매번 달라지지. MAC주소만 있는거.

라우터는 기본적으로 NIC이 2개 이상임.
NIC하나에는 공인ip를 받고, 다른거는 사설 ip
결국 라우터는 인터페이스가 2개지. LAN하나 WAN하나.경계선 역할인 셈

NAT : LAN과 WAN간에 네트워크 주소(public IP 주소 <-> private IP 주소)를 바꾼다. 옛날 시스템. 그림참조
NAPT : 포트까지 해준거.그림참조

port forwarding:
라우터야 특정 포트를 나한테만 걸어줄 수 잇어? 라고 하는것. 1:1로 바인딩 해놓은것.
private network에 가려져 있던 애를 콕 찝어서 연결 할 수 있게 하는 것.
그러면 바깥에서 안쪽으로 연결 되겠지.

port란?
소켓 파일이니까 소켓에 할당된 특별한 주소 같은 것.

소켓의 본질은 file

리눅스에서는 소켓을 파일 취급.

linux에선
test.txt
test라는 디렉토리
stdin, stdout, stderr
socket
모두 다 file임~~

윈도우는 file은 파일이고 소켓은 소켓임. 서로 다른 객체임.

포트란 소켓에 할당된 2byte짜리 숫자. 2^16까지 부여 가능.

0~1023까지의 포트 번호(well known port): 특정 서비스에 할당받은 특정 포트
http: 80번
https:443
ftp: 21
ssh: 22

https://www.youtube.com/watch?v=laBzCcF1414
socket의 본질은 file인데 file에다가 +알파(IP주소,port번호,종류..) 를 bind해서 씀
server단에서는 bind()가 있고 client단에서는 connect가 그 역할을 함.
따라서 port번호는 socket을 구성하고 socket은 process와 다른 컴퓨터와의 네트워킹을 규정하므로
port번호는 사실상 process 식별자 역할을 함
